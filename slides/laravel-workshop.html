Prerequisites

  • Composer - A Dependency Manager for PHP
  • NPM - (Node Package Manager) A Package Manager for Javascript
  • Local web server : Wamp, Xampp or LAMP stack
  • * make sure composer, npm and php is in your $PATH

    $ composer --version

    # should output composer version

    $ php -v|

    # should output php version

Installation

  • Download laravel installer using composer

    $ composer global require laravel/installer

  • Choose a folder and Initialize a new laravel project

    $ laravel new laravel-workshop

  • Create a virtual host to /public directory or run the following command to run the built in php dev server.

    $ php artisan serve|

    Then visit localhost:8000 or 127.0.0.1:8000

Configuration

  • env() - used in accessing .env file values (e.g. env('APP_NAME', 'Laravel'))
    • .env should only be accessed through configuration files and should not be included in application's source control
  • config() - used in accessing config files under /config directory
    • usage: config('config_file.config_name') (e.g. config('app.name'))

Routing and Middleware

  • process where http requests are routed to the code that handles them 
// routes/web.php
Route::get('/', function () {
  return view('welcome');
}); // route directly to view
Route::get('/test/index', 'TestController@index'}); // route to controller
Route::post('/test/store', 'TestController@store'})->name('test.store');// named
Route::show('/test/{id}/edit', 'TestController@edit'})->name('test.edit'); // with parameters

Route::show('/test/{id}/edit', 'TestController@index'})
  ->middleware('auth'); // Protected (only logged in)

route('test.store') // to access named routes in views and controller
route('test.index')->with('success', 'User added') // route with a session data
redirect()->route('test.edit', ['id' => 1]) // redirect to a route

Views & Scaffolding

  • Views contain the HTML and uses Blade Syntax. 
    <!-- routes/web.php -->
// passing data to view
Route::get('/test/{name}', function ($name) {
  $array = ['one', 'two', 'three'];

  return view('test.index', [
    'name' => $name,
    'array' => $array
  ]);
})->name('test.index');

    <!-- Sample: resources/views/test/index.blade.php -->
<html>
    <body>
        <h1>Hello, {{ $name }}</h1>
        @foreach ($iterable as $key => $value)
          <p>{{ $value }}</p>
        @endforeach
        <a href="{{ route('test.index') }}">Reload page</a>
    </body>
</html>
  • Generate login and register scaffolding and compile assets (// must be in virtual host to test)

    $ composer require laravel/ui:^1.0 --dev

    $ php artisan ui bootstrap --auth

    $ npm install && npm run dev

    $ php artisan migrate

Artisan

  • CLI that provides helpful commands that can assist in development

    $ php artisan list // show available commands

    $ php artisan make:controller TestController --resource

    $ php artisan make:migration CreateTestTable

    $ php artisan make:model Test

    $ php artisan make:model Test --migration --controller

    $ php artisan help [command]

  • tinker - allows you to interact with your entire Laravel application on the command line

    $ php artisan tinker

    >>> 2 + 2 //outputs 4

    >>> App::environment() // outputs current environment (e.g.'local')

    >>> $user = User::find(1) // get the user with the id of 1
