Prerequisites

  • Composer - A Dependency Manager for PHP
  • NPM - (Node Package Manager) A Package Manager for Javascript
  • Local web server : Wamp, Xampp or LAMP stack
  • * make sure composer, npm and php is in your $PATH

    $ composer --version

    # should output composer version

    $ php -v|

    # should output php version

Installation

  • Download laravel installer using composer

    $ composer global require laravel/installer

  • Choose a folder and Initialize a new laravel project

    $ laravel new laravel-workshop

  • Create a virtual host to /public directory or run the following command to run the built in php dev server.

    $ php artisan serve|

    Then visit localhost:8000 or 127.0.0.1:8000

Configuration

  • env() - used in accessing .env file values (e.g. env('APP_NAME', 'Laravel'))
    • .env should only be accessed through configuration files and should not be included in application's source control
  • config() - used in accessing config files under /config directory
    • usage: config('config_file.config_name') (e.g. config('app.name')) 
# .env
APP_NAME="Laravel Workshop"
APP_ENV=local
APP_DEBUG=true
APP_URL=http://laravel-workshop.test

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=laravel_workshop
DB_USERNAME=root
DB_PASSWORD=

MAIL_DRIVER=smtp

MAIL_PASSWORD=test1234
MAIL_ENCRYPTION=tls
MAIL_FROM_ADDRESS=test@gmail.com
MAIL_FROM_NAME="${APP_NAME}"

Routing and Middleware

  • process where http requests are routed to the code that handles them 
// routes/web.php
Route::get('/', function () {
  return view('welcome');
}); // route directly to view
Route::get('/users/index', 'UserController@index'}); // route to controller
Route::post('/users/store', 'UserController@store'})->name('users.store');// named
Route::get('/users/{id}/edit', 'UserController@edit'})->name('users.edit'); 
Route::put('/users/{id}/update', 'UserController@update'})->name('users.update'); 
Route::delete('/users/{id}/destroy', 'UserController@destroy'})->name('users.destroy'); // with parameters

Route::get('/users/{id}/edit', 'UserController@index'})
  ->middleware('auth'); // Protected (only logged in)

route('users.store') // to access named routes in views and controller
route('users.index')->with('success', 'User added') // route with a session data
redirect()->route('users.edit', ['id' => 1]) // redirect to a route with a parameter

Views & Scaffolding

  • Views contain the HTML and uses Blade Syntax. 
    <!-- routes/web.php -->
// passing data to a view
Route::get('/users/{name}', function ($name) {
  $array = ['one', 'two', 'three'];

  return view('users.index', [
    'name' => $name,
    'array' => $array
  ]);
})->name('users.index');

    <!-- Sample: resources/views/users/index.blade.php -->
<html>
    <body>
        <h1>Hello, {{ $name }}</h1>
        @foreach ($iterable as $key => $value)
          <p>{{ $value }}</p>
        @endforeach
        <a href="{{ route('users.index') }}">Reload page</a>
    </body>
</html>
  • Generate login and register scaffolding and compile assets (// must be in virtual host to test)

    $ composer require laravel/ui:^1.0 --dev // require laravel ui package

    $ php artisan ui bootstrap --auth // generate login and register and others

    $ npm install && npm run dev // install npm dependencies and compile assets

    $ php artisan migrate // run migrations

Artisan

  • CLI that provides helpful commands that can assist in development

    $ php artisan list // show available commands

    $ php artisan make:controller UserController --resource

    $ php artisan make:migration CreateUsersTable

    $ php artisan make:model User

    $ php artisan make:model User --migration --controller

    $ php artisan help [command]

  • tinker - allows you to interact with your entire Laravel application on the command line

    $ php artisan tinker

    >>> 2 + 2 //outputs 4

    >>> App::environment() // outputs current environment (e.g.'local')

    >>> $user = User::find(1) // get the user with the id of 1

Controllers

  • Controllers can group related request handling logic into a single class

    $ php artisan make:controller UserController --resource 

    <!-- app/Http/Controllers/UserController.php -->
use App\User;
class UserController extends Controller
{
    public function index()
    {
        $users = User::all();
        return view('users.index', ['users' => $users])
    }
    public function edit(User $user) // dependency injection
    {
        return view('users.edit', ['user' => $user]);
    }
    public function create() {}
    public function store(Request $request) {}
    public function edit($id) {}
    public function update(Request $request, $id) {}
    public function destroy($id) {}
}

    <!-- Sample: resources/views/users/index.blade.php -->
<html>
    <body>
        @foreach ($users as $user)
          <p>{{ $user->first_name }}</p>
        @endforeach
    </body>
</html>

    <!-- Sample: resources/views/users/edit.blade.php -->
<form action="{{ route('users.update', ['id' => $user->id]) }}">
  @csrf
  @method('PUT')

  <input type="first_name" value="{{ $user->first_name }}">
  <input type="submit">
</form>
